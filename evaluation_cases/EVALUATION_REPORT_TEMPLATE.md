## 1. 评测方法

### 1.1 测试案例

本次评测选取了 5 个代表性案例，覆盖不同类型和风格：

| 案例 ID | 案例名称 | 类型 | 特点 |

| Case 05 | 失眠少女与收梦猫 | 治愈日常 | 安静、情绪细腻、非对话驱动 |
| Case 03 | 旧教学楼的黑板留言 | 校园悬疑 | 悬疑但不恐怖、信息控制要求高 |
| Case 02 | 学校操场的外星信号 | 少年科幻 | 紧张感、渐进式揭示 |
| Case 04 | 地下机甲比赛 | 热血动作 | 压迫感、视觉冲击力 |

### 1.2 测试流程

每个案例执行以下流程：

**单 Agent Baseline**:
1. 输入故事种子 → 一次性生成完整内容
2. 输入 Revision 请求 → 修改内容

**Comic Agent Workflow**:
1. 输入故事种子 → Architect 生成角色卡、场景卡、剧本
2. 执行 `/director` → Director 生成分镜
3. 输入 Revision 请求 → Feedback Orchestrator 协调修改

### 1.3 评分标准

每个维度采用 1-5 分制：

| 维度 | 评分标准 |
|------|----------|
| **角色一致性** | 1=严重矛盾，5=完全一致 |
| **分镜稳定性** | 1=视觉锚点混乱，5=视觉连续性强 |
| **Revision 可控性** | 1=大范围改动，5=精准局部修改 |
| **信息控制** | 1=严重剧透，5=悬念保持完美 |
| **风格稳定性** | 1=风格跳变，5=风格统一 |
