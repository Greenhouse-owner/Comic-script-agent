## 输入

- `script.md`：章节剧本全文
- `selected_context.json`
- `storyboard_plan.json`
- 已读取的角色卡内容
- 已读取的场景卡内容
- `panel-director` 技能手册
- `reaction-polisher` 技能手册

## 输出格式

输出 Markdown 分镜脚本，保存为：

```text
chapters/{chapter_id}/storyboard.md
```

推荐格式：

```markdown
# ch01 分镜脚本

## 分镜规划摘要
- 页数：12
- 总格数：48
- 关键大格：第 2 页、第 9 页
- 节奏说明：前段铺垫，中段冲突推进，后段高潮与收束。

## 第 1 页
- 本页功能：建立场景
- 情绪目标：孤独、安静、略带神秘
- 格子数：4

### 格 1
- 大小：wide_large
- 景别：远景
- 机位：平视或俯视
- 画面：具体可画的画面描述
- 人物：出现人物及视觉锚点
- 视觉重点：本格焦点
- 对白/旁白：无或具体文本
- 音效：无或具体文本
```

## 硬约束

1. 必须遵循 `storyboard_plan.json` 的页数、格子数、格子大小和阅读顺序。
2. 不得改写剧本事实。
3. 每个角色首次出场必须体现角色卡视觉锚点。
4. 场景、道具、氛围必须参考场景卡。
5. 每格必须有明确视觉重点。
6. 对白和旁白必须来自剧本或忠实概括，不得新增改变剧情的信息。
7. 如果上下文缺失，要在开头 `## 注意事项` 中说明，不要擅自补设定。

## 失败处理

- 如果 `storyboard_plan.json` 缺少某些字段，按剧本顺序生成基础分镜，并在开头说明使用 fallback。
- 如果角色/场景上下文不足，仍生成分镜，但标记 `[上下文不足]`。
- 如果某段剧本不可画，标记 `[绘制困难]` 并给出可画替代方案。
