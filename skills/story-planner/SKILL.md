## 输入

- **故事梗概**：用户提供的 1-3 句话故事想法
- **用户选择记*（可选）：科幻/奇幻/现代/武侠等
录**（可选）：之前对话中用户做出的选择
- **目标风格*
## 输出格式

输出一个结构化的 Markdown 文档，包含：

```markdown
# 故事大纲

## 核心设定
- 世界观：...
- 主线冲突：...
- 核心主题：...

## 章节规划
### 第一章：标题
- 节拍：起始事件 → 冲突升级 → 小高潮
- 关键转折：...
- 预期页数：15-20

### 第二章：标题
...

## 关键人物
- 主角：姓名、身份、核心矛盾
- 对手：姓名、立场、动机
- 配角：...

## 关键场景
- 场景名、用途、氛围
```

## 硬约束

1. **必须形成可被后续章节继承的故事级事实源**
   - 世界观设定一旦确立，后续章节不能自相矛盾
2. **至少规划 3 章，最多 12 章**
   - 太少故事单薄，太多难以维护一致性
3. **每个关键人物必须有明确的动机**
   - 不能有"工具人"角色

## 失败处理

- **缺关键信息**：先保留最小故事骨架，标记`[待补充]`，不要伪装完整设定
- **用户描述模糊**：生成 2-3 个选择题让用户澄清（调用 `choice_designer` skill）
- **风格不确定**：默认使用"现代都市"风格
