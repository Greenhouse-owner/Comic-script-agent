## 输入

- **章节规划**：来自 story_planner 的输出
- **角色数据库**：`state/characters/` 文件夹下所有角色 `.md` 文件
- **场景数据库**：`state/environments/` 文件夹下所有场景 `.md` 文件
- **前置章节**（如果有）：确保连贯性

## 输出格式

输出 Markdown 剧本（保存为 `chapters/{chapter_id}/script.md`）：

```markdown
# 第一章：黑夜降临

## 场景 1：城市街头（夜晚）
**环境**：霓虹灯闪烁的老城区，雨后积水反射着招牌灯光。

林夜站在巷子口，黑色风衣被夜风吹起。他的左眼竖瞳在暗处发出微弱金光。

林夜（独白）："三年了，还是没有线索..."

突然，一个身影从巷子深处冲出——

## 场景 2：追逐
...
```

## 硬约束

1. **角色约束 — 数据库驱动**
   - 剧本中出现的每个角色必须在 `state/characters/` 文件夹中有对应的 `.md` 文件
   - 如需新角色，必须先暂停，调用 `character_db_manager` 的 create 操作创建角色文件
   - 角色描写不得违反其当前 Level 已确定的信息
   - 角色的外貌描写必须引用角色文件中的视觉锚点
   - Level 1 的角色：可以写外在行为和性格驱动的互动，不能写内心独白和深层动机
   - Level 2 的角色：可以写内心独白、动机驱动的选择、角色弧线转折
2. **场景约束 — 数据库驱动**
   - 剧本中出现的每个场景必须在 `state/environments/` 文件夹中有对应的 `.md` 文件
   - 如需新场景，必须先暂停，调用 `environment_db_manager` 的 create 操作创建场景文件
   - 场景描写必须引用场景文件中的 `key_props` 中的至少 2 个道具
3. **角色行为必须符合 personality 设定**
   - 参考角色文件的性格特质（Level 1）和致命缺陷（Level 2）
4. **visual_anchors 必须在首次登场时体现**
   - 如"黑色风衣"、"左眼竖瞳"必须在描述中出现
5. **场景转换要流畅**
   - 不能突然跳转到完全无关的地点

## 失败处理

- **角色文件不存在**：暂停，标记 `[❌ 未注册角色: XXX]`，自行调用 character_db_manager 创建
- **场景文件不存在**：暂停，标记 `[❌ 未注册场景: XXX]`，自行调用 environment_db_manager 创建
- **描写与数据库冲突**：标记 `[⚠️ 数据冲突: 数据库说X，剧本写了Y]`
- **角色 Level 不足以支撑剧情**：标记 `[⚠️ 需要升级: XXX 当前 Level N，本场景需要 Level M]`
