## 输入

- **文字剧本**：chapters/{chapter_id}/script.md
- **角色数据库**：`state/characters/` 文件夹下所有角色 `.md` 文件
- **场景数据库**：`state/environments/` 文件夹下所有场景 `.md` 文件

## 输出格式

输出分镜脚本（保存为 `chapters/{chapter_id}/storyboard.md`）：

```markdown
# 第一章分镜脚本

## 第 1 页（共 18 页）

### 格子 1（全景）
- **景别**：远景
- **机位**：鸟瞰，45度俯角
- **画面内容**：老城区街头全景，霓虹灯招牌林立，地面积水反射灯光
- **人物**：林夜站在巷口（剪影，占画面1/3）
- **气氛**：孤独、压抑
- **对白**：无

### 格子 2（中景）
- **景别**：中景
- **机位**：平视，略仰
- **画面内容**：林夜半身像，黑色风衣，银色短发被风吹起
- **视觉重点**：左眼竖瞳发出金色微光
- **对白**：林夜独白："三年了，还是没有线索..."

### 格子 3（特写）
- **景别**：特写
- **机位**：正面特写
- **画面内容**：林夜的左眼（竖瞳）特写，瞳孔中映出巷子深处的模糊身影
- **动态元素**：身影从模糊到清晰（动态暗示）
- **对白**：无
- **音效**：急促脚步声

...
```

## 硬约束

1. **视觉一致性 — 数据库驱动**
   - 角色的每次出场，分镜描述必须包含其角色文件中视觉锚点的至少 1 个特征
   - 角色首次出场的格子，必须包含全部视觉锚点
   - 场景的道具描绘必须参考场景文件中的 key_props
2. **每页 3-6 格**
   - 太少节奏拖沓，太多画面拥挤
2. **景别必须有变化**
   - 不能连续 3 格都是同一景别
   - 推荐比例：远景20% / 中景50% / 特写30%
3. **关键情节必须有特写**
   - 情感高潮、重要道具、转折点必须用特写或近景
4. **每格必须明确视觉重点**
   - 不能含糊地说"画面很乱"，要具体到哪个元素是焦点
5. **角色首次登场必须体现 visual_anchors**
   - 如"黑色风衣"、"左眼竖瞳"必须在对应格子中明确标注
6. **Level 约束**
   - Level 1 角色：可以画各种景别，必须符合视觉锚点，不画表现内心世界的特殊镜头
   - Level 2 角色：解锁所有表现手法，包括内心独白分镜、回忆杀、情感特写
   - Level 1 场景：可以展现场景全貌和道具细节
   - Level 2 场景：解锁象征性构图、情感渲染、场景演变对比

## 失败处理

- **剧本节拍不明确**：返回要求剧本补充转折点标记
- **角色动作不可画**：标记 `[绘制困难]`，提供替代方案
- **页数超标**（>25页/章）：压缩非关键场景
