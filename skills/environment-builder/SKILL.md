## 输入

- **场景名称**：如"老城区街头"
- **故事大纲**：用于确保场景符合世界观
- **用途说明**（可选）：该场景在故事中的作用
- **时间段**（可选）：如"深夜"、"黄昏"

## 输出格式

输出结构化 JSON（存入 `state/environments.json`）：

```json
{
  "name": "老城区街头",
  "location": "城市东区",
  "atmosphere": "压抑、阴冷、神秘",
  "time_of_day": "深夜23点",
  "key_props": [
    "霓虹灯招牌（闪烁的红蓝光）",
    "积水（反射灯光和人影）",
    "破旧垃圾桶（巷口标志物）"
  ],
  "description": "老城区的窄巷，两侧是密集的霓虹灯招牌，地面因刚下过雨而积水。灯光在水面上形成斑驳的倒影，空气中弥漫着潮湿和油烟的气味。"
}
```

## 硬约束

1. **key_props 至少 3 个**
   - 必须是具体可画的物件（"氛围好"不行，"破旧的霓虹灯招牌"可以）
2. **场景名称必须唯一**
   - 不能与已有场景重名
3. **atmosphere 必须是视觉可表现的**
   - 用具体的光线、色调、天气描述，不用抽象形容词
4. **description 至少 2 句话**
   - 要包含足够的视觉细节供分镜参考

## 失败处理

- **信息不足**：返回带 `[待补充]` 标记的半成品场景卡
- **与世界观冲突**：标记冲突点，提供修改建议
- **道具不够具体**：要求补充可画的细节描述
